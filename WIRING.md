# 绘图小车最终接线表

本文按“电源 → 电机/编码器 → LF04 → MAIXCAM → 板载外设 → 上电检查”的顺序接线。连接器脚号来自扩展板原理图，MCU 引脚与当前 Keil 工程一致。

## 1. 电源

扩展板同时使用三条电源轨，所有电源负极和模块 GND 必须共地。

| 电源 | 接入位置 | 用途 |
| --- | --- | --- |
| 电池正极 12 V | H2 或 H5 任一脚 | TB6612 `VM`、两个 MG310 电机 |
| 电池负极 | H1 任一脚 | 系统公共 GND |
| 5 V 降压正极 | H3 任一脚 | 核心板、OLED、TB6612 `VCC/STBY` |
| 5 V 降压负极 | H1 任一脚 | 系统公共 GND |
| 板载 3.3 V | H4 任一脚 | LF04、两个编码器 VCC |

- H1 四脚均为 GND；H2/H5 均为 `+12V`；H3 四脚均为 `+5V`；H4 四脚均为 `3V3`。
- 5 V 接入 H3 后，先测量 H4，确认约为 3.3 V，再接传感器。
- 核心板已由 5 V 供电时，不要把独立 3.3 V 降压模块并联到 H4，避免反向灌电。
- 12 V 绝不能接到 H3、H4、H6；电机电源线应使用比信号线更粗的导线。

## 2. 左电机与编码器 1

左轮使用 U12，对应代码中的电机 A 和编码器 1。

### U12 板端

| U12脚 | 板上信号 | 对应功能 |
| --- | --- | --- |
| 1 | AO1 | TB6612 电机 A 输出1 |
| 2 | GND | 左编码器地 |
| 3 | E1B | PB12，左编码器 B相 |
| 4 | E1A | PB23，左编码器 A相 |
| 5 | +5V | 不接编码器，线束中的 VCC 线退针并绝缘 |
| 6 | AO2 | TB6612 电机 A 输出2 |

### MG310 PH2.0-6P 到 U12

| MG310脚 | MG310功能 | 接到哪里 |
| --- | --- | --- |
| 1 | 电机− | U12.6 / AO2 |
| 2 | 编码器 VCC | H4 / 3V3 飞线 |
| 3 | 编码器 A相 | U12.4 / E1A |
| 4 | 编码器 B相 | U12.3 / E1B |
| 5 | 编码器 GND | U12.2 / GND |
| 6 | 电机+ | U12.1 / AO1 |

MG310 原生线序与 U12 板端线序不同，不能按相同脚号直通，必须制作交叉线束。

## 3. 右电机与编码器 2

右轮使用 U9，对应代码中的电机 B 和编码器 2。

### U9 板端

| U9脚 | 板上信号 | 对应功能 |
| --- | --- | --- |
| 1 | BO1 | TB6612 电机 B 输出1 |
| 2 | GND | 右编码器地 |
| 3 | E2B | PB5，右编码器 B相 |
| 4 | E2A | PB4，右编码器 A相 |
| 5 | +5V | 不接编码器，线束中的 VCC 线退针并绝缘 |
| 6 | BO2 | TB6612 电机 B 输出2 |

### MG310 PH2.0-6P 到 U9

| MG310脚 | MG310功能 | 接到哪里 |
| --- | --- | --- |
| 1 | 电机− | U9.6 / BO2 |
| 2 | 编码器 VCC | H4 / 3V3 飞线 |
| 3 | 编码器 A相 | U9.4 / E2A |
| 4 | 编码器 B相 | U9.3 / E2B |
| 5 | 编码器 GND | U9.2 / GND |
| 6 | 电机+ | U9.1 / BO1 |

若架空测试时某轮正 PWM 反向，只交换该轮的两根电机线，即 U12.1/U12.6 或 U9.1/U9.6；不要交换编码器电源和地线。

## 4. TB6612 与 MCU 内部对应

这些网络已由 PCB 连接，不需要飞线，仅用于排查。

| 功能 | 左轮/电机 A | 右轮/电机 B |
| --- | --- | --- |
| PWM | PA28 / PWMA | PB20 / PWMB |
| 方向1 | PA13 / AIN1 | PB9 / BIN1 |
| 方向2 | PB26 / AIN2 | PB7 / BIN2 |
| 电机输出 | U12.1、U12.6 | U9.1、U9.6 |
| 编码器 A相 | PB23 / E1A | PB4 / E2A |
| 编码器 B相 | PB12 / E1B | PB5 / E2B |

TB6612 模块的 `VM`、`VCC`、`STBY` 已分别接板载 `+12V`、`+5V`、`+5V`，不需要额外飞线。

## 5. LF04 四路循迹

LF04 安装在驱动轮轴前方。以小车车头方向为基准，O1 在车辆左侧，O4 在车辆右侧。

| LF04 | 扩展板 | MCU |
| --- | --- | --- |
| VCC | H4 / 3V3 | 3.3 V |
| GND | H6.10 | GND |
| O1 最左 | H6.8 / C1 | PA31 |
| O2 左中 | H6.7 / C2 | PA12 |
| O3 右中 | H6.6 / C3 | PB8 |
| O4 最右 | H6.1 / C8 | PB10 |

H6.9 是 `+5V`，禁止接 LF04 VCC。

## 6. MAIXCAM PRO

MAIXCAM 单独可靠供电并与 MSPM0 共地，改用 MSPM0 的 UART2，不能使用扩展板串口座的 3.3 V 给摄像头供电。

| MAIXCAM | MSPM0 |
| --- | --- |
| A28 / UART2_TX | PB16 / MSPM0 UART2_RX |
| A29 / UART2_RX | PB15 / MSPM0 UART2_TX，可选 |
| GND | 公共 GND |
| 电源 | 独立可靠供电 |

串口参数为 `115200、8N1`，TX 必须交叉连接到对方 RX。

## 7. 串口屏

串口屏占用 MSPM0 的 UART1，并使用 H3 的 5 V 供电。已确认屏幕 UART 信号为 3.3 V TTL，可以直接连接 PA8/PA9；CN6.1 的 3.3 V 不给屏幕供电。

| 串口屏 | 扩展板/MCU |
| --- | --- |
| TX | CN6.4 / PA9 / UART1_RX |
| RX | CN6.3 / PA8 / UART1_TX |
| GND | CN6.2 / 公共 GND |
| VCC | H3 / +5V |

串口参数为 `115200、8N1`。启动按钮的弹起事件必须为 `printh 31 0D 0A;`，即发送十六进制字节 `31 0D 0A`。固件只在 `WAIT START` 状态接受该命令，运行中重复点击不会急停或重新启动。

## 8. 板载 OLED、中键和调试串口

OLED 和中键已由 PCB 连接，无需飞线。

| 功能 | MCU |
| --- | --- |
| OLED SCL | PB2 |
| OLED SDA | PB3 |
| OLED电源 | +5V |
| 中键 | PB24，按下接地、低电平有效 |

UART0/CN5 继续保留为调试串口：

| CN5脚 | 功能 | MCU/连接 |
| --- | --- | --- |
| CN5.1 | 3V3 | 仅供低功耗 3.3 V 设备 |
| CN5.2 | GND | 公共地 |
| CN5.3 | UART0_TX | PA10 |
| CN5.4 | UART0_RX | PA11 |

## 9. 安全上电顺序

1. 暂不连接电机、LF04、编码器和 MAIXCAM。
2. 万用表确认 `+12V`、`+5V`、`3V3` 对 GND 没有短路。
3. 单独调整 5 V 降压模块到约 5.00 V。
4. 5 V 接入 H3，确认 H4 输出约 3.3 V。
5. 接入 LF04 和两个编码器的 3.3 V 线，确认 U9.5/U12.5 的 5 V 没有进入编码器。
6. 手动转动车轮，用示波器确认四路 A/B 高电平约为 3.3 V。
7. 接入 MAIXCAM 独立电源和公共地，确认其 TX 已接 PB16。
8. 屏幕使用 H3 的 5 V 和公共地，TX/RX 分别交叉接 PA9/PA8。
9. 架空小车，最后接入 12 V 电机电源。
10. 确认正 PWM 时左右轮均朝车头方向行驶。
11. 确认前进时左右编码器的软件累计值均为正。
12. 全部通过后再落地运行。

## 固定对应关系

- U12 为左轮，U9 为右轮。
- 编码器 1 为左轮，编码器 2 为右轮。
- LF04 O1 位于车体左侧。
- 两个编码器使用 H4 的 3.3 V，不使用 U9.5/U12.5 的 5 V。
- 外部激光不连接 MSPM0，不属于当前软件控制范围。
